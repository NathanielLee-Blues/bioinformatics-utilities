# Bioinformatics Utilities

## Project overview

This repository contains four lightweight Python command-line tools for inspecting common bioinformatics inputs. They operate on a real *Escherichia coli* reference genome, 1,000-read FASTQ subsets and a filtered VCF generated by the companion variant-calling workflow.

The programs are deliberately small enough for their parsing and validation logic to be read directly. They are intended for transparent checks and learning rather than as replacements for mature production software.

## Included tools

| Tool | Purpose |
|---|---|
| `fasta_stats.py` | Calculates sequence counts, lengths and GC percentage |
| `fastq_basic_qc.py` | Calculates read-length and mean Phred-quality summaries |
| `vcf_summary.py` | Counts PASS records, SNPs and indel/complex variants |
| `reverse_complement.py` | Returns the reverse complement of a DNA sequence |

## Results from the included inputs

### FASTA summary

The REL606 reference contains one sequence of 4,629,812 bases with a GC percentage of 50.77%.

### FASTQ summaries

| File | Reads | Total bases | Mean read length | Mean Phred quality |
|---|---:|---:|---:|---:|
| R1 subset | 1,000 | 127,430 | 127.43 | 35.22 |
| R2 subset | 1,000 | 131,415 | 131.41 | 35.55 |

These mean scores indicate high average base quality in the subsets, but a single mean cannot show positional quality decline, adapter contamination or the distribution of low-quality bases.

### VCF summary

The filtered VCF contains 407 PASS records: 353 SNPs and 54 indel or complex variants, all on contig `CP000819.1`.

## Preparing the inputs

The preparation script copies the reference and filtered VCF from the variant-calling project and creates 1,000-read FASTQ subsets:

```bash
bash scripts/prepare_inputs.sh
```

## Running the tools

```bash
python3 src/fasta_stats.py \
  data/inputs/reference/ecoli_rel606_reference.fasta \
  -o results/fasta_stats.csv

python3 src/fastq_basic_qc.py \
  data/inputs/reads/SRR2584866_R1_1000_reads.fastq \
  -o results/fastq_qc_R1.csv

python3 src/fastq_basic_qc.py \
  data/inputs/reads/SRR2584866_R2_1000_reads.fastq \
  -o results/fastq_qc_R2.csv

python3 src/vcf_summary.py \
  data/inputs/variants/SRR2584866_filtered_variants.vcf \
  -o results/vcf_summary.csv

python3 src/reverse_complement.py ATGCCGTA
```

The reverse-complement example returns:

```text
TACGGCAT
```

## Testing

Run the standard-library test suite with:

```bash
python3 tests/run_tests.py
```

The tests check expected values for the included real-data inputs and basic sequence behaviour.

## Interpretation and limits

The tools make file structure and summary calculations explicit, which is useful for checking assumptions and understanding how common formats are parsed. They load and process data using simple Python logic, so they are not optimised for very large files and do not provide the breadth of validation available in FastQC, MultiQC, BCFtools, SAMtools or established parsing libraries.
